# Task: Auth Backend

## Description
Implement JWT authentication endpoints using djangorestframework-simplejwt. Create register, login, token refresh, and profile endpoints. Implement custom permission classes (IsStudent, IsTeacher) for role-based access control.

## Acceptance Criteria
- [ ] POST /api/auth/register/ — Creates user with role selection (student/teacher), validates fields
- [ ] POST /api/auth/login/ — Returns JWT access (15min) + refresh (7 days) tokens
- [ ] POST /api/auth/token/refresh/ — Refreshes access token
- [ ] GET /api/auth/profile/ — Returns authenticated user profile
- [ ] PUT /api/auth/profile/ — Updates profile (full_name, avatar, grade_class)
- [ ] Custom User serializer with role, full_name, grade_class fields
- [ ] IsStudent permission class — allows only role='student'
- [ ] IsTeacher permission class — allows only role='teacher'
- [ ] Password validation on registration
- [ ] grade_class required only for students

## Technical Details
- Use simplejwt's TokenObtainPairView for login, customized to include user data in response
- Register endpoint: custom view (not ViewSet) — creates user + returns tokens
- Profile: RetrieveUpdateAPIView with IsAuthenticated permission
- Permission classes in accounts/permissions.py
- URL configuration in accounts/urls.py, included in config/urls.py under /api/auth/
