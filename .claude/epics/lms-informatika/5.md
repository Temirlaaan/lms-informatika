# Task: Course Content Backend

## Description
Implement DRF API for course content: sections, topics, lessons, and progress tracking. Create ViewSets with nested serializers (section→topics→lesson), a topic completion endpoint, and a student progress summary endpoint.

## Acceptance Criteria
- [ ] GET /api/courses/sections/ — List all published sections (with topic count and student progress %)
- [ ] GET /api/courses/sections/:id/ — Section detail with nested topics (each showing completion status for current student)
- [ ] GET /api/courses/topics/:id/ — Topic detail with nested lesson content (HTML, video_url, images)
- [ ] POST /api/courses/topics/:id/complete/ — Mark topic as completed for current student (creates TopicProgress)
- [ ] GET /api/courses/progress/ — Student's overall progress (completed topics per section, percentage)
- [ ] Nested serializers: SectionSerializer includes topics, TopicSerializer includes lesson
- [ ] Only published sections/topics returned for students
- [ ] Teacher can see unpublished content
- [ ] TopicProgress unique_together enforced (no duplicate completions)
- [ ] URLs configured under /api/courses/

## Technical Details
- courses/serializers.py: SectionListSerializer, SectionDetailSerializer, TopicDetailSerializer, LessonSerializer, TopicProgressSerializer
- courses/views.py: SectionViewSet (list, retrieve), TopicViewSet (retrieve + complete action), ProgressView
- courses/urls.py: Router-based URLs
- Progress calculation: count completed TopicProgress for student per section / total topics in section
- Permissions: IsAuthenticated for all; complete action requires IsStudent
