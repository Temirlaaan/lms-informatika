# Task: Auth Frontend

## Description
Implement authentication UI and infrastructure. Create AuthContext for JWT token management, Axios interceptor for automatic token refresh, Login and Register pages in Kazakh, ProtectedRoute component for role-based routing, and layout components (PublicLayout, StudentLayout, TeacherLayout with sidebar navigation).

## Acceptance Criteria
- [ ] AuthContext: stores tokens in localStorage, provides login/logout/register functions, tracks current user
- [ ] Axios instance with JWT interceptor: auto-attaches access token, auto-refreshes on 401
- [ ] Login page (/login): username + password form, Kazakh UI text, error handling
- [ ] Register page (/register): username, password, full_name, role selector, grade_class (conditional), Kazakh UI
- [ ] Landing page (/): course description, "Кіру" and "Тіркелу" buttons
- [ ] ProtectedRoute component: checks auth + role, redirects to /login or role-appropriate dashboard
- [ ] App.tsx with React Router v6: public routes, /student/* routes, /teacher/* routes
- [ ] PublicLayout: header with nav links
- [ ] StudentLayout: sidebar with navigation (Басты бет, Бөлімдер, Бағалар), main content area
- [ ] TeacherLayout: sidebar with navigation (Басты бет, Контент, Тесттер, Журнал, Оқушылар), main content area
- [ ] All UI text in Kazakh language
- [ ] Responsive design (mobile + desktop)

## Technical Details
- AuthContext in context/AuthContext.tsx — useContext hook pattern
- Axios instance in api/axios.ts with request/response interceptors
- API functions in api/auth.ts (login, register, getProfile, updateProfile, refreshToken)
- TypeScript interfaces in types/index.ts (User, AuthTokens, LoginRequest, RegisterRequest)
- Tailwind CSS for styling with configured color palette
- React Router v6 nested routes with Outlet for layouts
