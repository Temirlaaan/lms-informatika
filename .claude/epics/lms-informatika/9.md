# Task: Quiz & Grades Backend

## Description
Implement DRF API for the quiz system (start, submit, scoring) and grades (student journal, teacher gradebook, statistics). Quiz submission includes automatic scoring and Grade record creation. Correct answers must be hidden until quiz is completed.

## Acceptance Criteria
- [ ] GET /api/quizzes/topic/:topic_id/ — Get quiz for a topic (questions + choices, WITHOUT is_correct)
- [ ] POST /api/quizzes/:quiz_id/start/ — Start a quiz attempt (checks max_attempts limit, creates QuizAttempt)
- [ ] POST /api/quizzes/:quiz_id/submit/ — Submit answers, auto-score, create Grade. Returns results with correct answers
- [ ] GET /api/quizzes/attempts/ — List student's quiz attempts
- [ ] GET /api/quizzes/attempts/:id/ — Detailed attempt result (questions, student answers, correct answers, points)
- [ ] GET /api/grades/my/ — Student's grades grouped by section
- [ ] GET /api/grades/journal/ — Teacher gradebook (all students × all sections)
- [ ] GET /api/grades/student/:id/ — Individual student grades (teacher only)
- [ ] GET /api/grades/statistics/ — Class statistics (average scores, completion rates per section)
- [ ] Scoring logic: compare selected_choices with is_correct, calculate earned_points/total_points, percentage
- [ ] Grade auto-calculation: ≥85%→5, ≥70%→4, ≥50%→3, <50%→2
- [ ] Attempt limit enforcement (default 3)
- [ ] Server-side time_limit validation on submit
- [ ] is_correct field excluded from Choice serializer during active quiz

## Technical Details
- quizzes/serializers.py: QuizSerializer (hides is_correct), QuestionSerializer, ChoiceSerializer, QuizAttemptSerializer, StudentAnswerSerializer, QuizResultSerializer (shows is_correct)
- quizzes/views.py: QuizViewSet (retrieve by topic + start/submit actions), AttemptViewSet (list, retrieve)
- grades/serializers.py: GradeSerializer, GradebookSerializer, StatisticsSerializer
- grades/views.py: StudentGradesView, TeacherGradebookView, StudentDetailView, StatisticsView
- Scoring in quizzes/services.py or in submit action: iterate questions, check answers, sum points
- Grade creation: after scoring, create/update Grade for student+section with grade_value
- Permissions: Quiz actions require IsStudent; Gradebook/journal require IsTeacher
