# Task: Teacher Frontend

## Description
Build all teacher-facing pages: dashboard with class statistics, content management (CRUD for sections/topics/lessons), quiz management (CRUD for quizzes/questions/choices), gradebook showing all students, and individual student detail page. All UI in Kazakh.

## Acceptance Criteria
- [ ] /teacher/dashboard — Class statistics: average scores per section, total students, completion rates, charts/summary cards
- [ ] /teacher/content — Content management:
  - List sections with edit/delete buttons and "add section" button
  - Section form: title, description, order, icon, is_published
  - Topic list within section, with edit/delete and "add topic"
  - Topic form: title, order, is_published
  - Lesson editor within topic: HTML content textarea, video_url input
- [ ] /teacher/quizzes — Quiz management:
  - List quizzes grouped by section/topic
  - Quiz form: title, description, time_limit, passing_score, max_attempts, is_published
  - Question list within quiz, with edit/delete and "add question"
  - Question form: text, question_type selector, points, order
  - Choice list within question: text, is_correct toggle, add/remove choices
- [ ] /teacher/gradebook — Full gradebook: table of students × sections, showing grade values with color coding
- [ ] /teacher/students/:id — Student detail: profile info, per-section grades, quiz attempt history
- [ ] API integration: api/teacher.ts (CRUD for all entities), api/grades.ts (getJournal, getStudentGrades, getStatistics)
- [ ] All forms have validation and error messages in Kazakh
- [ ] Confirmation dialogs for delete operations
- [ ] All UI text in Kazakh

## Technical Details
- Pages in pages/teacher/: Dashboard.tsx, ContentManager.tsx, QuizManager.tsx, Gradebook.tsx, StudentDetail.tsx
- Reusable components: DataTable, FormModal, ConfirmDialog, StatCard
- CRUD pattern: list view with table → click to edit (modal or inline) → save via API
- Teacher API: /api/teacher/sections/, /api/teacher/topics/, etc. — standard REST CRUD
- Nested management: section → topics → lessons; quiz → questions → choices
- Statistics: fetch from /api/grades/statistics/, display as summary cards

## Dependencies
- [ ] Task 4 (Auth Frontend) — layout, auth context, axios
- [ ] Task 5 (Course Backend) — course data structure
- [ ] Task 9 (Quiz & Grades Backend) — quiz structure, grades API, teacher endpoints
