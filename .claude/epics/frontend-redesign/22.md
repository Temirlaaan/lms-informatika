# Task: Setup shadcn/ui Foundation

## Description
Install all required npm packages, configure path aliases, create the `cn()` utility, rewrite `index.css` with shadcn CSS variable theming system (light + dark), and add Inter font. This is the foundation task that all other tasks depend on.

## Acceptance Criteria
- [ ] `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react` installed
- [ ] `@/` path alias configured in `vite.config.ts` (resolve.alias) and `tsconfig.app.json` (compilerOptions.paths, baseUrl)
- [ ] `src/lib/utils.ts` created with `cn()` function (clsx + twMerge)
- [ ] `src/index.css` rewritten with full shadcn CSS variable set for `:root` (light) and `.dark` (dark) selectors
- [ ] Inter font added via Google Fonts link in `index.html`
- [ ] `index.html` updated with `class="light"` on `<html>` element
- [ ] Existing pages still render without errors after changes
- [ ] `npx tsc -b` passes

## Technical Details
- **vite.config.ts**: Add `resolve: { alias: { "@": path.resolve(__dirname, "./src") } }`
- **tsconfig.app.json**: Add `"baseUrl": "."`, `"paths": { "@/*": ["./src/*"] }`
- **CSS variables**: background, foreground, card, popover, primary, secondary, muted, accent, destructive, border, input, ring (each with DEFAULT and foreground)
- **Color palette**: Blue primary (#2563EB → hsl), green accent, slate neutrals
- Keep existing `--color-primary` and `--color-accent` as aliases during migration

## Files Affected
- `frontend/vite.config.ts`

- `frontend/src/lib/utils.ts` (new)
- `frontend/src/index.css`
- `frontend/index.html`
